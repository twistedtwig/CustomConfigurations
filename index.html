Customconfigurations

a C# configuration Manager handler to allow for multiple clients settings to exists and be easily accessed via custom configuration settings

Download this project as a .zip file Download this project as a tar.gz file

Welcome to GitHub Pages.

Custom configurations is designed to reduce the repeative lines of code needed to get configuration settings into your application. If there are 5 or 50 settings there doesn't need any extra code to get your settings. It allows the application to simplify the use of configuration settings. A simple but effective use is to have an object with N number of properties, such as:

$ cd your_repo_root/repo_name
$ git fetch origin
$ git checkout gh-pages


public class MyAppSettings{    public string Name { get; set; }    public string Location { get; set; }    public bool ShouldDoX { get; set; }    public int minValue { get; set; }}



Then use the configuration loader to populate the values:




MyAppSettings appSettings = new CustomConfigurations.Config().Create();
